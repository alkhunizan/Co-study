# P3 Channel Shortlist — Halastudy beta distribution

> **Purpose:** Get 30+ named humans on the beta list by end of Day 6. Pick ONE channel, work it, fall back only if it under-delivers. Don't multi-channel at the start — it dilutes signal.

## Recommended channel order

### Primary — WhatsApp groups Aziz already runs ⭐
**Why first:** warmest audience, zero distribution cost, fastest path to 30+ names, audience already trusts the sender.
**Risk:** existing group context may not match Halastudy's target student cohort. If the groups are mostly biohacking/longevity/marketing peers (per CLAUDE.md global), some won't be the demand-side but might know students.
**Volume target:** 15-20 names from this alone if 3 groups are seeded.

### Backup 1 — X (Twitter) seed thread + DM follow-ups
**Why second:** uses an existing audience, costs only time. Thread can run in Arabic OR English (try Arabic first per CLAUDE.md "Arabic-first" positioning).
**Risk:** weak conversion to "yes I'd join a beta" from cold timeline replies. Better for awareness than conversion.
**Volume target:** 5-10 names if the thread gets any reach.

### Backup 2 — University Telegram / r/saudiarabia / Discord study servers
**Why third:** colder, requires more upfront context. Useful only if 1+2 under-deliver. r/saudiarabia leans general-interest; targeted university Telegram groups (KAUST, KFUPM, KSU) are higher leverage.
**Volume target:** 5+ names, slow burn.

## Seed-message templates

Each template is ≤3 sentences. Goal is a yes/maybe/no reply, **not** a pitch. Send variations — don't copy-paste identical messages, especially in WhatsApp where it reads as spam.

### Template A — WhatsApp / DM, Arabic (primary)

```
شباب، أبني تطبيق صغير لطلاب الخليج — غرفة دراسة مع كاميرا، بدون
كلام، تكفي إن صاحبك جالس يذاكر معك على الشاشة عشان تركّز.
لسه نسخة بيتا. أبغى ٥ أشخاص يجربون. تجرب؟
```

### Template B — WhatsApp / DM, English (mixed-cohort groups)

```
Building a small thing for Gulf students — a study room where you
join with camera on but mic off, body-doubling style. No timers,
no chat, just "we're studying together." Looking for ~5 beta
testers. Want in?
```

### Template C — X / Twitter seed thread (Arabic, primary)

```
أبني هلا ستدي — غرفة دراسة هادئة:
كاميرا مفتوحة. مايك مقفل. تذاكر مع صاحبك على الشاشة بدون كلام.
بيتا قريب. اللي يبي يجرب، رد بـ "أبي" أو بصمة ✋.
```
Then follow up with each reply via DM using Template A.

### Template D — University Telegram (last resort, formal)

```
طلاب [KAUST / KFUPM / KSU]، أبني تطبيق دراسة جماعي للطلاب — كاميرا
على، صوت مقفل، تذاكرون مع بعض على الشاشة. مفتوح لبيتا مغلق.
اللي يبي يجرب، يرد هنا أو يكلمني خاص.
```

## Decision rule

- **End of Day 5:** total yes-to-beta ≥ 15 → P3 trending green. Continue same channels.
- **End of Day 5:** total yes-to-beta < 8 → switch primary channel OR cut the spike scope to landing-page-validation only.
- **End of Day 6:** total < 30 → P3 yellow at best for the Day-7 go/no-go.

## What "green" looks like
≥30 named humans with WhatsApp/IG/X handles by end of Day 6. Doesn't matter if they're 30 yeses and 0 maybes or 25 yeses and 5 maybes — names are names.

## What "yellow" looks like
15-29 names. Means the channel works but slowly. Day 7 keeps the wedge but extends the beta-recruitment window into Week 2.

## What "red" looks like
<15 names by end of Day 6. The channel doesn't carry the product. Day 7 go/no-go should downgrade or scope-cut.

## Tips
- Don't gate replies on a sign-up form. WhatsApp handle = name on list. Form friction kills mid-funnel.
- Reply within 2 hours of a "yes." Speed signals reality.
- Save every Arabic "أبي" / "أبغى" / "ممكن" reply — those are Day-6's verbatim distribution quotes.
- If a "yes" turns into a 10-message back-and-forth with no commitment, move on. Half-yeses don't count as names on the list.
